# 🎬 短视频分析工作流设计思路

## 📋 需求分析

### 用户原始需求
用户希望创建一个短视频分析工作流，能够评估以下几个核心维度：
1. ✅ **故事连贯性** - 情节是否逻辑清晰
2. ✅ **画面风格统一性** - 视觉风格是否一致
3. ✅ **文案台词吸引力** - 语言表达是否有吸引力
4. ✅ **画质清晰度** - 视频技术质量如何

### 需求进一步拆解
基于专业视频制作经验，我将分析维度扩展为：

#### 🎯 **内容层面**
- **故事连贯性**：情节逻辑、转折设置、主线副线关系
- **受众匹配度**：目标群体适配性、情感共鸣点

#### 🎨 **视觉层面** 
- **画面风格统一性**：色彩搭配、构图质量、视觉一致性
- **画质清晰度**：分辨率、噪点控制、压缩质量

#### ✍️ **文案层面**
- **台词吸引力**：语言表达力、开头Hook设计、行动召唤
- **音频质量**：语音清晰度、背景音乐匹配度

#### 🔧 **技术层面**
- **制作工艺**：剪辑技术、特效使用、音频处理
- **平台适配**：格式规范、移动端优化

#### 📊 **商业层面**
- **传播潜力**：分享意愿、讨论话题性
- **用户体验**：观看舒适度、完播意愿

## 🏗️ 工作流架构设计

### 设计原则

#### 1. **模块化设计**
- 每个分析维度独立成节点
- 便于维护和功能扩展
- 支持并行和串行混合处理

#### 2. **智能分支逻辑**
- 根据视频时长选择不同分析策略
- 短视频(< 30秒)：注重快节奏、开头吸引力
- 长视频(≥ 30秒)：注重内容深度、结构完整性

#### 3. **数据驱动分析**
- 参数提取节点自动获取视频基础信息
- 基于结构化数据进行后续专业分析
- 减少重复信息获取，提高分析精度

#### 4. **专业化评估**
- 每个节点都有专门的系统提示词
- 模拟真实的专业团队分工协作
- 最终由"总监级"角色生成综合建议

### 节点设计逻辑

#### 🎬 **开始节点**
```yaml
输入变量设计：
- video_files: 支持多文件上传，便于批量分析
- video_type: 预设8种常见类型，提供针对性分析
- analysis_requirements: 自定义需求，个性化分析重点
```

#### 📋 **基本信息分析节点**
```yaml
角色设定：专业视频内容分析师
分析任务：
- 提取视频技术参数（时长、分辨率、格式）
- 概述主要内容和主题
- 初步质量评估
启用视觉：支持GPT-4o视觉分析能力
```

#### 🔍 **参数提取节点**
```yaml
提取参数：
- duration (number): 用于时长判断
- resolution (string): 画质分析依据
- theme_type (string): 内容类型识别
- has_person (boolean): 人物出镜判断
- has_text (boolean): 字幕存在性
- music_type (string): 音频风格识别
```

#### ⏱️ **条件分支节点**
```yaml
判断逻辑：duration ≥ 30秒
- True路径：长视频深度分析策略
- False路径：短视频快节奏分析策略
```

#### 🎨 **专业分析节点群**
每个分析节点都有专门的角色设定：

1. **内容分析师**：故事逻辑、节奏把控
2. **视觉设计师**：构图、色彩、画质
3. **文案策划师**：语言表达、音频质量
4. **技术专家**：剪辑工艺、后期制作

#### 📊 **变量聚合节点**
```yaml
聚合策略：
- 收集所有分析结果
- 统一变量命名规范
- 为最终评估提供完整数据
```

#### 📈 **综合评估节点**
```yaml
角色设定：顶级短视频制作总监
评估维度：
- 五维度评分系统（100分制）
- 优先级改进建议（紧急/重要/长期）
- 具体执行方案
- 成功案例参考
```

#### 📄 **模板转换节点**
```yaml
输出格式：
- Markdown格式专业报告
- 结构化信息展示
- 视觉友好的排版
- 可操作的建议清单
```

## 🔧 技术实现细节

### Dify节点使用

#### **LLM节点配置**
```yaml
模型选择: GPT-4o (支持视觉分析)
温度设置: 
  - 技术分析: 0.1 (客观准确)
  - 创意评估: 0.3 (适度创新)
视觉功能: 启用(用于画面分析)
```

#### **参数提取节点**
```yaml
提取方式: LLM推理
数据类型: 混合(string/number/boolean)
必需参数: duration, resolution, theme_type
可选参数: music_type
```

#### **条件分支节点**
```yaml
判断条件: 数值比较 (≥ 30)
分支逻辑: 真/假二元选择
变量引用: {{#parameter-extractor.duration#}}
```

#### **变量聚合节点**
```yaml
聚合方式: 手动选择变量
变量映射: 
  - long_video_analysis → 长视频分析结果
  - short_video_analysis → 短视频分析结果
  - visual_analysis → 视觉分析结果
  - script_analysis → 文案分析结果
  - technical_analysis → 技术分析结果
```

### Prompt工程

#### **系统提示词设计原则**
1. **角色明确性**：每个节点都有清晰的专业角色定位
2. **任务具体性**：详细列出分析维度和评估标准
3. **输出规范性**：要求结构化、可操作的建议
4. **上下文连贯性**：引用前序节点的分析结果

#### **示例Prompt结构**
```yaml
系统角色: "你是一名资深的[专业角色]，专门[专业领域]"
分析维度: 
  1. 维度1 - 子项目a, b, c
  2. 维度2 - 子项目d, e, f
输出要求: "请提供专业、详细的分析报告"
```

### 变量传递机制

#### **数据流设计**
```mermaid
graph LR
    A[用户输入] --> B[基本分析] --> C[参数提取]
    C --> D[条件判断] 
    D --> E[专业分析群]
    E --> F[结果聚合] --> G[综合评估] --> H[格式化输出]
```

#### **变量命名规范**
- 使用语义化命名：`video_files`, `analysis_requirements`
- 节点ID采用数字序列：`1736412001000` 系列
- 聚合变量使用描述性名称：`visual_analysis`, `technical_analysis`

## 📊 评估体系设计

### 评分标准

#### **五维度评分系统**
1. **故事连贯性**(100分)
   - 情节逻辑性(30分)
   - 转折合理性(25分)
   - 结构完整性(25分)
   - 主题一致性(20分)

2. **画面风格统一性**(100分)
   - 色彩协调性(25分)
   - 构图质量(25分)
   - 视觉一致性(25分)
   - 美学效果(25分)

3. **文案台词吸引力**(100分)
   - 开头吸引力(30分)
   - 语言表达力(25分)
   - 情感共鸣度(25分)
   - 行动召唤效果(20分)

4. **画质清晰度**(100分)
   - 分辨率质量(30分)
   - 压缩优化(25分)
   - 噪点控制(25分)
   - 色彩还原(20分)

5. **技术制作水平**(100分)
   - 剪辑技术(30分)
   - 音频处理(25分)
   - 特效使用(20分)
   - 后期工艺(25分)

### 建议分级

#### **三级改进建议**
1. **🚨 紧急改进项**
   - 影响基本观看体验的问题
   - 技术性错误或明显缺陷
   - 需要立即处理

2. **⭐ 重要优化项**
   - 显著提升质量的改进点
   - 用户体验优化建议
   - 短期内可实施

3. **🎯 长期提升方向**
   - 战略性改进建议
   - 品牌建设相关
   - 需要持续投入

```yaml
处理逻辑: 对视频列表进行批量分析
```


#### 3. **实时评分系统**
```yaml

响应速度: 优化为快速评估模式
输出简化: 关键指标和核心建议
```

#### 4. **行业定制版本**
```yaml
教育类: 增加知识传递效果评估
商业类: 增加转化潜力分析
娱乐类: 增加病毒传播要素评估
```

### 技术优化方向

#### 1. **成本优化**
- 使用更小的模型处理基础分析
- 重要节点使用高精度模型
- 条件判断减少不必要的LLM调用

#### 2. **速度优化**
- 并行处理独立的分析节点
- 预设模板减少重复生成
- 缓存常用分析结果

#### 3. **准确性优化**
- 多模型交叉验证
- 人工反馈循环优化
- 历史数据训练fine-tuning

## 💡 设计心得

### 成功要素

1. **用户需求理解**：从简单需求出发，深入挖掘专业需求
2. **模块化设计**：每个功能独立，便于维护和扩展
3. **专业化角色**：模拟真实工作场景，提高分析质量
4. **数据驱动**：结构化参数提取，支撑精准分析
5. **可操作建议**：不仅评估问题，更提供解决方案

### 挑战与解决

#### **挑战1：视频分析的复杂性**
**解决方案**：分层分析 + 专业分工
- 基础信息 → 技术参数 → 专业分析 → 综合评估

#### **挑战2：主观评估的标准化**
**解决方案**：评分体系 + 角色设定
- 100分制量化评估 + 专业角色模拟

#### **挑战3：不同类型视频的差异化需求**
**解决方案**：条件分支 + 类型判断
- 时长判断选择策略 + 视频类型针对性分析

#### **挑战4：输出结果的实用性**
**解决方案**：分级建议 + 执行方案
- 紧急/重要/长期分级 + 具体操作步骤
