# 提示词库 (Prompts)

这个目录用于存放日常工作中使用的提示词文件，帮助提高工作效率。

## 目录结构

```
prompts/
├── README.md              # 本说明文件
├── work/                  # 工作相关提示词
│   ├── meeting.md        # 会议相关
│   ├── email.md          # 邮件相关
│   └── report.md         # 报告相关
├── coding/               # 编程相关提示词
│   ├── code_review.md    # 代码审查
│   ├── bug_fix.md        # Bug修复
│   └── documentation.md  # 文档编写
├── learning/             # 学习相关提示词
│   ├── explanation.md    # 概念解释
│   └── summary.md        # 总结归纳
└── creative/             # 创意相关提示词
    ├── writing.md        # 写作
    └── brainstorm.md     # 头脑风暴
```

## 使用指南

### 1. 文件命名规范
- 使用小写字母和下划线：`file_name.md`
- 文件名要能反映提示词的用途
- 使用 `.md` 格式便于阅读和编辑

### 2. 提示词格式
每个提示词文件建议包含：
- **标题**：简明的提示词名称
- **用途**：说明这个提示词的使用场景
- **提示词内容**：具体的提示词模板
- **示例**：使用示例（可选）
- **标签**：便于分类查找

### 3. 示例格式
```markdown
# 提示词标题

## 用途
描述这个提示词的使用场景...

## 提示词
```
你是一个专业的...
请帮我...
```

## 示例
输入：...
输出：...

## 标签
#工作 #效率 #模板
```

## 快速开始

1. 在相应的分类目录下创建新的 `.md` 文件
2. 按照上述格式编写提示词
3. 在需要时复制粘贴使用

## 贡献指南

- 提示词要实用且经过验证
- 保持格式统一
- 添加清晰的说明和示例
- 使用合适的标签便于检索
