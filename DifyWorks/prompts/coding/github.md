## 本地覆盖云端仓库 提示词

按照如下步骤执行任务

### 步骤一
1. 比对DifyWorks项目和用户名是 benjaminhou2 的 github 账号的DifyWorks项目
2. 提供本地比云端多的文件详细清单；
3. 提供本地和云端都存在的文件，但本地文件版本高于云端版本的文件信息清单；
4. 提供云端比比对多的文件详细清单；
5. 提供本地和云端都存在的文件，但本地文件版本低于云端版本的文件信息清单；


### 步骤二

完成步骤一的文件对比，必须等待我选择接下来选择如下四个方案中的一个方案去这个，这个环节必须由我来确认，且需要采用二次确认的方式，两次确认 如果选择的是同一方案，则按照该方案执行。

如果两次确认选择的不是同一方案，则重新进行方案选择，并继续进行二次确认。

### 方案1. 完全以本地文件为基准
1. 删除本地没有但云端仓库中有的任何文件；
2. 本地有但云端没有的文件，提交到仓库；
3. 本地和服务器仓库均有的文件，以本地文件为准覆盖服务器文件


### 方案2.完全以服务器端仓库文件为准
1. 删除本地有但服务器端仓库中没有的任何文件；
2. 本地没有但云端有的文件，更新同步到本地；
3. 本地和服务器仓库均有的文件，以服务器端文件为准覆盖本地文件


### 方案3.合并文件后更新
1. 如果本地没有但云端仓库中有的任何文件，先从服务器仓库同步到本地，同步后保障服务器仓库的文件清单是本地端的子集；
2. 如果仓库中的文件版本高于本地文件的版本，则将仓库的文件更新至本地；
3. 确认完成服务器文件同步本地之后，将本地项目全部文件同步服务器仓库；


### 方案4.智能合并更新
1. 对于冲突文件，优先保留内容更丰富、版本更新的文件；
2. 对于新增文件，全部保留；
3. 确保关键文件（如配置文件、重要文档）的完整性。
